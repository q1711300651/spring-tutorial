ACL (Access Control List)

ACL сервисы предостовляються в отдельном архиве spring-security-acl-xxx.jar.

Каждый доменный обьект имеет собственный список ACL, и этот список контролирует кто имеет право обратиться к обьетку.

Spring Security позволяет реализовать в приложении
    1. Способ для эффектривного плучения ACL сущностей для всех доменных обьектов
    2. Способ для проверки ползователя для доступа к доменным обьектам, перед вызовом метода
    3. Способ для проверки прав пользователя для доступа с обьектом после вызова метода

Использование таблиц в бд для ACL

    1. ACL_SID - позволяет получить уникальный идентификатор для пользователя или прав в системе
    2  ACL_CLASS - позволяет получить уникальный идентификатор для любого класса доменноно обьекта в системе
    3. ACL_OBJECT_IDENTITY - сохраняет информацию по каждому уникальному обьекту в системе.
    4. ACL_ENTRY - сохраняет индивидуальные права присвоенные на пользователе


Интерфейсы
   Acl - кадждый доменный обьект имеент только один Acl, который содержит AccessControlEntry
   AccessControlEntry - содержит информацию о прав доступа
   Permission - права доступа представляют как неизменяемая битовая маска, предостовляющая простые способы для получение
                информации
   Sid - security identity - связывает Acl модуль с пользователеми приложения
   ObjectIdentity - предстовление доменного обьека в модуле Acl
   AclService - сервис для получение Acl по ObjectIdentity
