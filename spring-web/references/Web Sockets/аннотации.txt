@MessageMapping
    Аннотация @MessageMapping поддреживаеться контроллером и используеться для связывание методов с адресатами
    Поддерживаеться такие ant-стиля адрессаты "/foo/**", "/foo*" и с параметрами адреса "/foo/{id}"

    Доступные аргументы @MessageMapping методов:
        Message - сообщение
        @Payload - анотированный аргумент, (так и не понял что это)
        @Header аргумент, доступ к хедеру
        карта @Headers
        MessageHeaders
        @DestinationVariable доступ к значением шаблона адрессата  "/foo/{id}"
         java.security.Principal - проверка логгина пользователя, в момент "рукопожатия" WebSocket

@SendTo
    Указывает адресат результата метода/иначе возвращает по адресату @MessageMapping

@SubscribeMapping
    По умолчанию значение что возрващают @SubscribeMapping методы, отправляються на прямую к клиенту, огибая
    брокер. Это полехно, скажем для реализации выгрузки данных для UI, что начинает загружаться.
